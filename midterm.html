Midterm Exam

Q1:

Which command line utility would you use to determine the following:

  1. The spatial extent of a GML file?

    2. ogrinfo

  2. The spatial reference system of a raster file in the format of a GeoTIFF?

    3. gdalinfo

Q2:

What combination of OGC Service and Request (e.g. SERVICE=WMS&REQUEST=GetFeatureInfo) would you use to perform the following?

  1. Determine the spatial extent of an available layer from a Web Map Service.

    2. SERVICE=WMS&REQUEST=GetCapabilities

  2. Obtain a list of coverages from a Web Coverage service.

    3. SERVICE=WCS&REQUEST=DescribeCoverage

  3. Determine what file formats are supported by a Web Feature Service for the delivery of available data types (i.e. layers).

    4. SERVICE=WFS&REQUEST=GetCapabilities

  4. Retrieve a map image from a Web Map Service.

    5. SERVICE=WMS&REQUEST=GetMap

  5. Retrieve data from an available coverage from a Web Coverage Service.

    6. SERVICE=WCS&REQUEST=GetCoverage

Q3:

Use the cs2cs command line utility to convert the following latitude-longitude coordinates (WGS84) to UTM, Zone 13N, NAD83 coordinates.

  1. 35.682180 North Latitude -105.939670 East Longitude (New Mexico State Capitol)

    2. 414970.04     3949105.23 0.00

  2. 34.891983 North Latitude -107.930153 East Longitude (El Malpais National Monument)

    3. 232227.14     3864983.25 0.00

Q4:

Perform the following WMS GetCapabilities request and answer the following questions:

http://neowms.sci.gsfc.nasa.gov/wms/wms?version=1.1.1&service=WMS&request=GetCapabilities

  • What is the name of the service?

    • OGC:WMS (NASA Earth Observations)

  • What file formats are supported by the GetMap request?

    • image/png

    image/jpeg

  • What are the names of three of the layers included in the service?

    • Cloud Fraction

    Cloud Particle Radius

    Cloud Water Content

Q5:

Compose a GetMap request for the WMS referenced in Question 4 that includes the following characteristics:

  1. JPEG image format

  2. 1200 pixels wide (you will need to calculate the height based upon the aspect ratio of the bounding box)

  3. Bounding Box (EPSG:4326): Min X = -128 East Longitude, Min Y = 21.5 North Latitude, Max X = -62 East Longitude, Max Y = 54.5 North Latitude

  4. Layer to be mapped = “MOD_LSTD_CLIM_M”

Include in your answer both the complete WMS GetMap request and the resulting map image that is returned.

http://neowms.sci.gsfc.nasa.gov/wms/wms?
VERSION=1.1.1&
SERVICE=WMS&
REQUEST=GetMap&
BBOX=-128,21.5,-62,54.5&
LAYERS=MOD_LSTD_CLIM_M&
WIDTH=1200&
HEIGHT=600&
SRS=EPSG:4326&
FORMAT=image/jpeg&
STYLES=

Q6:

From the XML GetCapabilities returned by the following WFS request answer the following questions:

http://services.nationalmap.gov/arcgis/services/WFS/transportation/MapServer/WFSServer?request=GetCapabilities&service=WFS

  • What is the title for this service?

    • WFS_transportation

  • What file format(s) are supported by this service’s GetFeature request?

    • text/xml

    text/gml

  • What is the DefaultSRS or the FeatureType named “WFS_transportation:Interstate”?

    • urn:ogc:def:crs:EPSG:6.9:4326

  • What is the WGS84BoundingBox of the FeatureType named “WFS_transportation:Interstate”?

    • Lower: -158.10419299999992     21.277850000000058

    Upper: -67.781173999999908     49.002374000000088

Q7:

From the XML GetCapabilities returned by the following WCS request answer the following questions:

http://sdf.ndbc.noaa.gov/thredds/wcs/hfradar_uswc_500m?request=GetCapabilities&version=1.0.0&service=WCS

  • What is the description of the v coverage?

    • v m s-1 true surface_northward_sea_water_velocity

  • How many coverages are available from this service?

    • Four (4) coverages are available. They are “v”, “u”, “DOPy”, and “DOPx”.

Q8a:

Formulate a complete DescribeCoverage request for the v coverage for the WCS service referenced in Question 7.

http://sdf.ndbc.noaa.gov/thredds/wcs/hfradar_uswc_500m?service=WCS&version=1.0.0&request=DescribeCoverage&COVERAGE=v

Q8b:

From the returned XML document answer the following questions:

  • What is the spatial domain for the ‘v‘ coverage?

    • -122.59346771240234     37.45548629760742

    -122.0468978881836     38.13872528076172

  • What file formats are available for ‘v‘ data delivered by this service?

    • GeoTIFF

    GeoTIFF_Float

    NetCDF3

  • What SRS(s) are supported by this service for requested data delivery?

    • urn:ogc:def:crs:OGC:1.3:CRS84

    EPSG:0 [Latitude_Longitude]
